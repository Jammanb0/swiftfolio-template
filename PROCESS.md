# 개발 과정

이 프로젝트는 **Claude Code(AI 코딩 에이전트)를 활용해 기획부터 구현, 배포 설정까지 진행**했습니다. 아래는 그 과정의 핵심 요약입니다.

## 프로젝트 개요

- **목적**: 프론트엔드 개발자용 포트폴리오 사이트를 만들면서, 다른 사람도 그대로 가져다 쓸 수 있는 오픈소스 템플릿으로 분리
- **산출물**: 이 레포(`swiftfolio-template`, 재사용 가능한 템플릿)와 `Jammanb0.github.io`(이 템플릿으로 만든 실제 개인 사이트) 2개

## AI를 활용한 작업 방식

- 디자인 방향(Toss 스타일 UI), 기능 요구사항(마크다운 지원, 필터, 유튜브 임베드 등), 협업 규칙(새 의존성/git push는 사전 승인 필요)을 구체적으로 지시
- 매 기능 단위로 브라우저에서 직접 실행·검증하고, UI 디테일(드롭다운 화살표 정렬, 태그 필터 AND/OR 로직 등)을 피드백하며 반복 개선
- `CLAUDE.md`에 프로젝트 컨벤션과 의사결정 기록을 남겨, AI가 매 세션 동일한 기준으로 작업하도록 관리
- 커밋을 기능 단위로 잘게 나눠 히스토리를 리뷰 가능하게 유지 (`feat(scope): ...` 컨벤션)

## 기술 스택

- **React 19 + TypeScript + Vite** — 빌드 도구/프레임워크
- **vanilla-extract** — zero-runtime CSS-in-TS 디자인 시스템 (Emotion 대신 단일 스타일링 솔루션으로 통일)
- **Framer Motion + GSAP** — 마이크로 인터랙션/페이지 전환(Framer Motion), 히어로 스크롤 연출(GSAP)
- **react-router-dom** — 라우팅 + GitHub Pages SPA 대응
- **react-markdown + remark-gfm** — 프로젝트 상세 설명 마크다운 렌더링
- **gh-pages** — GitHub Pages 배포

## 주요 의사결정

- **vanilla-extract 단일 채택**: 스타일링 라이브러리를 하나로 통일해 유지보수 부담을 줄임
- **Fork 기반 배포 유지**: GitHub의 "Template repository" 기능 대신 일반 Fork 방식을 유지 — Fork 수가 레포에 공개적으로 표시되어 템플릿의 실사용 임팩트를 보여주기 좋다고 판단
- **마크다운 + 커스텀 유튜브 문법**: 새 의존성을 최소화하면서(react-markdown만 추가) 프로젝트 설명에 표/이미지/유튜브 영상까지 자유롭게 넣을 수 있도록 ` ```youtube ` 코드블록 컨벤션을 직접 설계
- **라우트 단위 코드 스플리팅**: 마크다운 렌더러처럼 무거운 의존성이 상세 페이지에만 필요하다는 점을 파악해 `React.lazy`로 분리, 초기 번들 크기를 절반 이상 축소
